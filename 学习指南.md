## 📖 详细学习目录

### 【v1】第 1 章：基础 CRUD（增删改查）

**文件夹**: `v1-crud/`

**核心知识点**:
```
┌─────────────────┐
│   Controller    │  ← 处理 HTTP 请求 + 业务逻辑
├─────────────────┤
│   Repository    │  ← 数据库操作（JPA）
└─────────────────┘
```

**学到的内容**:
- ✅ `@RestController` - RESTful API 控制器
- ✅ `@RequestMapping` - 路由配置
- ✅ `@GetMapping` `@PostMapping` `@PutMapping` `@DeleteMapping`
- ✅ `@RequestBody` `@PathVariable` `@RequestParam` - 参数接收
- ✅ `JpaRepository` - JPA 数据访问
- ✅ `@Entity` `@Table` `@Id` - 实体类定义
- ✅ `Result<T>` - 统一返回格式
- ✅ `@ExceptionHandler` - 全局异常处理

**文件结构**:
```
v1-crud/
├── controller/
│   └── UserController.java      # 所有 CRUD 接口
├── repository/
│   └── UserRepository.java      # 数据库操作接口
├── entity/
│   └── User.java                # 用户实体
└── common/
    ├── Result.java              # 统一返回
    └── GlobalExceptionHandler.java  # 异常处理
```

**如何运行**:
```bash
cd D:/code/code/status/v1-crud
mvn spring-boot:run
# 访问：http://localhost:18080/api/users
```

### 【v2】第 2 章：Service 层（三层架构）

**文件夹**: `v2-service/`

**核心知识点**:
```
┌─────────────────┐
│   Controller    │  ← 处理 HTTP 请求（只负责接收和返回）
├─────────────────┤
│    Service      │  ← 业务逻辑（验证、计算、事务）NEW!
├─────────────────┤
│   Repository    │  ← 数据库操作
└─────────────────┘
```

**学到的内容**:
- ✅ `@Service` - 服务层注解
- ✅ 接口 + 实现类模式（`UserService` + `UserServiceImpl`）
- ✅ `@Transactional` - 事务控制
- ✅ 业务逻辑与 Controller 分离
- ✅ 代码复用和可测试性

**文件结构**:
```
v2-service/
├── controller/
│   └── UserController.java      # 只处理 HTTP
├── service/
│   ├── UserService.java         # 服务接口（NEW!）
│   └── impl/
│       └── UserServiceImpl.java # 服务实现（NEW!）
├── repository/
│   └── UserRepository.java
├── entity/
│   └── User.java
└── common/
    ├── Result.java
    └── GlobalExceptionHandler.java
```

**如何运行**:
```bash
cd D:/code/code/status/v2-service
mvn spring-boot:run
# 访问：http://localhost:18080/api/users
```

**与 v1 的区别**:
| 对比项 | v1-crud | v2-service |
|--------|---------|------------|
| Controller 职责 | HTTP + 业务逻辑 | 只处理 HTTP |
| Service 层 | ❌ 无 | ✅ 有 |
| 业务逻辑位置 | Controller | Service |
| 事务控制 | ❌ 无 | ✅ `@Transactional` |
| 代码复用 | ❌ 难 | ✅ 易 |
| 适合场景 | 简单 CRUD | 复杂业务 |

## 📝 测试命令（所有版本通用）

```powershell
# 1. 查询所有用户
curl "http://localhost:18080/api/users"

# 2. 新增用户
curl -X POST "http://localhost:18080/api/users" `
  -H "Content-Type: application/json" `
  -d "{\"username\":\"zhangsan\",\"age\":25}"

# 3. 根据 ID 查询
curl "http://localhost:18080/api/users/1"

# 4. 更新用户
curl -X PUT "http://localhost:18080/api/users/1" `
  -H "Content-Type: application/json" `
  -d "{\"id\":1,\"username\":\"lisi\",\"age\":28}"

# 5. 删除用户
curl -X DELETE "http://localhost:18080/api/users/1"

# 6. 模糊查询
curl "http://localhost:18080/api/users/search/keyword?name=zhang"

# 7. 年龄范围查询
curl "http://localhost:18080/api/users/age?min=20&max=30"
```
